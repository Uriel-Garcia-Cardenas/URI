Enterprise Continuum

Clasifica arquitecturas y proporciona artefactos de soluciones en un continuo de desarrollo. Permite comprender la evolución de la arquitectura en una organización. Se divide en dos partes:

  • Architecture Continuum: Describe cómo se desarrollan las arquitecturas, desde arquitecturas base como TOGAF hasta arquitecturas específicas de la industria.
  • Solutions Continuum: Representa la implementación de la arquitectura con componentes y soluciones reutilizables.

Architecture Development Method (ADM)

Un proceso estructurado que guía la creación y evolución de la arquitectura empresarial. Define fases específicas para la planificación, diseño, implementación y gestión de la arquitectura.

Architecture Repository

Un repositorio centralizado para almacenar artefactos de arquitectura, modelos y mejores prácticas.

Architecture Capability Framework

Conjunto de recursos y habilidades necesarias para desarrollar y gestionar la arquitectura empresarial dentro de la organización.

Security Architecture

Protege los datos y la infraestructura de la organización:

  • Logical Security Architecture: Protege datos en aplicaciones web y redes mediante firewalls y seguridad en capas.
  • Physical Security Architecture: Protección física de datos y usuarios mediante infraestructura y mecanismos de seguridad.

Management Architecture

Incluye modelos para mejorar la gestión de la organización:

  • Procurement Architecture: Modela el proceso de adquisición de insumos médicos.
  • Business Architecture with Technology Architecture Support: Usa capas de servicio, gestión y datos para mejorar la operatividad hospitalaria.

Network Architecture

Modelo de red con capas Core, Distribution y Access para garantizar estabilidad y eficiencia.

Industry Architecture

Mejora la relación con los pacientes a través de plataformas digitales para gestionar quejas y mejorar servicios.
